# deploy — Helm chart (ADR-020 delivery contract)

`grizzly-platform`'s Flux `HelmRelease` for this app renders `./deploy` (see `kubernetes/apps/grizzly-gameservers/` over there). This is where the bot/agent workload, its ServiceAccount, and app-level wiring live.

`templates/deployment.yaml` is the bot/agent Deployment. `templates/service.yaml` is the ClusterIP Service fronting the bot's in-game agent endpoint (port `9360`, the `@Gary` reverse loop). `templates/externalsecrets.yaml` syncs the bot/agent credentials from OpenBao into namespace-local Secrets via the `openbao` `ClusterSecretStore` — toggle with `externalSecrets.enabled` in `values.yaml`. It renders five ExternalSecrets: `discord-bot` and `ollama-api` (under `grizzly-platform/gameservers/{discord,ollama}`), `grizzly-gameservers-db` and `grizzly-gameservers-s3` (foundation-store grants under `grizzly-platform/stores/grizzly-gameservers`, for the Postgres registries and world backups), and `grizzly-gameservers-ingame` (the shared in-game agent bearer token under `grizzly-platform/gameservers/ingame`). The Deployment consumes those Secrets and never talks to OpenBao directly (the `game-servers` egress leash blocks `10.0.0.200:8200`).

Agones is packaged as a standalone gated `HelmRelease` rather than a Helm dependency of this chart (see [ADR-001](../docs/decisions/001-agones-packaging.md)); it and the `cluster/` guardrails + Kyverno carve-out are wired separately, not through this chart.
